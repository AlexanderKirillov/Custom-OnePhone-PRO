О рековери (recovery)

Рековери позволяет войти в «режим восстановления» (recovery mode). При помощи рековери вы можете выполнить сброс устройства к заводским настройкам, установить обновление, установить модификацию прошивки, сделать полную резервную копию установленной прошивки и восстановить ее, установить кастомную прошивку и даже кастомное ядро и т.д. Рековери устанавливается в специальный системный раздел внутренней памяти телефона и не зависит от работоспособности прошивки телефона. Установка рековери выполняется при помощи компьютера. Обычно для установки рековери требуется, чтобы телефон уже имел рут или разблокированный загрузчик. Рековери адаптируются для каждого аппарата отдельно.
Различается два основных вида рековери:
1. Clockworkmod Recovery (CWM)
В этом рековери управление и навигация происходит при помощи аппаратных клавиш регулировки громкости (громче – шаг вверх, тише – шаг вниз) и кнопки питания, которая выполняет роль подтверждения (своеобразный аналог компьютерному Enter’у)
2. Team Win Recovery Project (TWRP)
В этом рековери управление и навигация происходит при помощи прикосновений и виртуальных кнопок. Так же TWRP обладает немного расширенным функционалом, по сравнению с CWM, и более удобен в использовании.
О том, как установить рековери для вашей модели телефона, порядок установки рековери, требования и прочее вы можете узнать в теме обсуждения прошивки вашего телефона на форуме 4PDA.RU

Установка через Clockworkmod (CWM) recovery
1) Для начала давайте ознакомимся с чем мы будем иметь дело:

ClockworkMod recovery (CWM) - модифицированный неофициальный recovery, который выпускается для большинства устройств, работающих под управлением Android. Обладает гораздо более широкими возможностями, чем родной recovery. На некоторых устройствах устанавливается вместо родного recovery, на других устройствах устанавливается параллельно. Позволяет:
-Устанавливать неофициальные прошивки
-Устанавливать дополнения и исправления
-Подключаться к ПК по USB в режиме съёмного накопителя
-Подключиться к ПК по USB в режиме ADB
-Создавать резервную копию как всей прошивки, так и отдельных её частей (система, настройки, установленные программы)
-Восстанавливать данные из ранее созданной резервной копии
-Сбрасывать данные к заводским настройкам (Wipe – data/factory reset), очищать кеш раздел (wipe cache), очищать далвик-кеш (wipe Dalvik-cache), очищать статистику работы аккумулятора (wipe battery stats)
-Форматировать и создавать разделы на карте памяти

Если вы не можете включить телефон и загрузить android, вы всегда можете восстановить через CWM ранее созданную резервную копию со всеми приложениями и настройками.

2) Навигация по меню:
-кнопка увеличения громкости - вверх.
-кнопка уменьшения громкости - вниз.
-кнопка включения/выключения - выбор пункта.
-кнопка контекстного меню - листать вниз.
-кнопка HOME (домой) - выбор пункта.
-кнопка назад - назад.

3) Первым делом всегда делайте BACKUP: (Резервное копирование системы)
- Запускаем CWM recovery
- Выбираем "Backup & restore"
- Выбираем "backup"
- Выбираем "YES"
- Выбираем Reboot system now (перезагрузка)
После этого на карте памяти появится полная резервная копия в папке clockworkmod/backup. Название резервной копии включает время и дату её создания, но название можно давать произвольное, разумеется латиницей без пробелов.

4) После этого мы можем непосредственно перейти к установке мода/прошивки/ядра/получению ROOT прав и т.д.
а) Закидываете .zip файл на флешку
б) В рекавери выбираете пункт install zip from sdcard
в) Затем выбираете пункт choose zip from sdcard
г) Находите на SD карте файл и нажимаете на него
д) Из множества пунктов Nо выбираете один единственный "Yes"
е) После завершения установки выбираете нажимаете один раз назад (чтобы выйти в главное меню) и нажимаете reboot system now.

5) В случае неудачной установки, или если вам не понравилось, то восстанавливаем BACKUP:
- Входим в CWM Recovery;
- Выбираем "backup & restore"
- Выбираем "restore"
- Выбираем сохранённую ранее резервную копию. Она должна находиться на карте памяти в папке clockworkmod/backup.
- Подтверждаем восстановление, выбираем "YES". Пункт с подтверждением обычно не первый по списку, это нормально
- По окончании процесса выбираем "Reboot system now" (перезагрузка).
6) Дополнительно, для тех, кто хочеть покапаться в CWM recovery:
-Reboot system - перезагрузка устройства
-apply update.zip from sdcard - установка файла update.zip, находящийся в корне карты памяти sdcard\
-wipe data/factory reset - полный сброс настроек и данных, при этом очищаются разделы /data и /cache во внутренней памяти устройства и очищается системная папка ".android_secure" на карте памяти
-wipe cache partition - очистка раздела /cache во внутренней памяти устройства
-install zip from sdcard - установка zip файла с карты памяти

1. chooze zip from sdcard - выбрать zip файл на карте памяти для установки
2. toggle signature veritification - вкл/выкл проверку подписи файла
3. toggle script assert - вкл/выкл скрипт утверждения

-backup and restore - создание и восстановление резервной копии

 1. Backup - создание резервной копии всех разделов в памяти устройства
2. Advanced Backup - создание резервной копии определённого раздела в памяти устройства
3. Restore - восстановление всех разделов из выбранной резервной копии
4. Advanced Restore - восстановление определённого раздела из выбранной резервной копии

-mounts and storage - монтирование и накопитель

 1. mount /system - монтирование папки системы
2. unmount /data - отмонтирование папки с пользовательскими данными
3. unmount /cache отмонтирование папки с кешем
4. mount /sdcard - монтирование карты памяти
5. mount /sd-ext - монтирование линукс раздела с карты памяти (если есть)
6. format boot - очистка загрузочного раздела
7. format system - очистка системного раздела
8. format data - очистка раздела с пользовательскими данными и настройками
9. format cache - очистка кеш раздела
10. format sdcard - форматирование карты памяти
11. format sd-ext - форматирование линукс раздела на карте памяти
12. mount USB storage - подключение к компьютеру в режиме съёмного накопителя

-advanced - дополнительные
1. Reboot Recovery - перезагрузка recovery
2. Wipe Dalvik Cache - очистка Dalvik кеша
3. Wipe Battery Stat - сброс статистики работы аккумулятора
4. Report Error - сообщить об ошибке
5. Key Test - проверка клавиш, при нажатии на кнопку, выдаёт её код
6. Restart adbd - перезагрузка ADB сервера
7. Process dump - Дамп
